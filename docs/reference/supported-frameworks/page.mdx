# Supported Frameworks

Temps auto-detects your framework by inspecting files in your repository and applies the appropriate build configuration. This page lists every supported framework with its detection signals and build behavior. {{ className: 'lead' }}

---

## How framework detection works {{ anchor: true, id: 'how-framework-detection-works' }}

When you create a project or push code, Temps scans the repository root (or the configured app directory) for known files. The first matching pattern determines the framework preset.

Detection priority:

1. **Dockerfile** ‚Äî If a `Dockerfile` exists, Temps uses a direct Docker build (highest priority)
2. **Framework-specific config files** ‚Äî `next.config.js`, `vite.config.ts`, `nuxt.config.ts`, etc.
3. **Language-specific files** ‚Äî `package.json`, `requirements.txt`, `go.mod`, `Cargo.toml`, etc.
4. **Static fallback** ‚Äî If only `index.html` is present, Temps serves it as a static site

All non-Dockerfile builds use [Nixpacks](https://nixpacks.com/) ‚Äî a build system that generates a Docker image from your source code without requiring a Dockerfile.

### Package manager detection

Nixpacks detects your package manager from lockfiles:

| Lockfile | Package manager |
|----------|----------------|
| `bun.lockb` or `bun.lock` | Bun |
| `pnpm-lock.yaml` | pnpm |
| `yarn.lock` | Yarn |
| `package-lock.json` | npm |

### Node.js version

Nixpacks reads the Node.js version from (in order):

1. `.node-version` file
2. `engines.node` in `package.json`
3. Default: latest LTS

Detected from (in order):

1. `.python-version` file
2. `[tool.poetry.dependencies]` in `pyproject.toml`

## Docker {{ anchor: true, id: 'docker' }}

If a `Dockerfile` is present in the repository root (or the configured app directory), Temps uses it directly instead of Nixpacks.

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY . .
EXPOSE 3000
CMD ["node", "server.js"]
```

The `EXPOSE` directive determines the port Temps routes traffic to. If not specified, Temps uses the environment or project `exposed_port` setting (default: 3000).

### Multi-stage builds

Multi-stage Dockerfiles work as expected:

```dockerfile
FROM node:20 AS builder
WORKDIR /app
COPY . .
RUN npm ci && npm run build

FROM node:20-alpine
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
EXPOSE 3000
CMD ["node", "dist/server.js"]
```

## Static sites {{ anchor: true, id: 'static-sites' }}

If your repository contains only static files (HTML, CSS, JavaScript) with no framework configuration:

| Detection signal | Behavior |
|-----------------|----------|
| `index.html` in root, no `package.json` | Served directly by Temps |
| `package.json` with build output (`dist/`, `build/`, `out/`) | Built with Nixpacks, output served as static |

Static sites are served by Temps' built-in web server. No Docker container is needed.

## Override detection {{ anchor: true, id: 'override-detection' }}

If Temps detects the wrong framework, override it in project settings:

1. Open your project in the dashboard
2. Go to **Settings**
3. Change the **Framework Preset** to the correct one
4. Optionally set a custom **Build Command** and **Start Command**

You can also set a custom install command, build command, or app directory for monorepos.
